# CLAUDE.md — EHBA Hardcourt Bike Polo ruleset (IT)

Context for Claude Code. Read this before editing anything in this repo.

## What this repo is

The official **Italian translation** of the EHBA Hardcourt Bike Polo ruleset,
revision `230712` (2023). It is published two ways from a single source:

1. a **PDF** exported from iA Writer using a custom theme, and
2. a **scrollable web version** on GitHub Pages, generated automatically.

## Golden rule — single source of truth

- **Edit ONLY the iA Writer source Markdown**: `EHBA-ruleset-rev230712-IT.md`
  (repo root — verify the exact filename; it is the file with iA Writer syntax).
- **Never hand-edit the web output.** The Pages version is a *derived artifact*
  produced at build time from the source. Editing it directly is always wrong; the
  next build overwrites it.
- The `2018 / rev200324` file is a historical archive. Do not update it to 2023.

## The three non-obvious traps (READ THIS)

These caused real bugs during translation. Do not rediscover them the hard way.

### 1. Clause numbers are generated by CSS, not written in the text

The visible numbering (`3.1.2.1 …`) is produced by **CSS counters** in
`assets/css/style-ruleset.css` from the **list-nesting depth** — NOT from the
digits in the Markdown. In the source, a clause looks like `3121. text`, but that
literal number is consumed as a list marker and regenerated by CSS. Therefore:

- **Nesting depth is everything.** If a clause is at the wrong indent level, its
  number comes out wrong even if the typed digits look right.
- When adding/removing/moving clauses, match the **tab depth** to the intended
  hierarchy. The typed number is a courtesy for the human editor in iA Writer.

### 2. iA Writer nests tab-indented lists; kramdown/GitHub does NOT

The source uses **tabs** and **multi-digit manual markers** (`311.`, `3121.`).
iA Writer nests these correctly. kramdown (GitHub Pages' engine) and CommonMark
decide nesting by **marker width**, so they *flatten* the tab-indented lists into
plain text and the numbering collapses. This is why there is a build-time
preprocessor (`build_page.py`) that:

- rebuilds list nesting from **tab depth**,
- normalises every ordered marker to `1.` (safe: CSS regenerates the real number),
- converts iA syntax (`{{TOC}}`, `+++`, `# Title [sectionN]`, `[x][sectionN]`).

**Implication for edits:** keep using tabs + the iA conventions in the source.
Do not "fix" the lists into spaces or single-level numbering — the preprocessor
depends on the tab structure.

### 3. The bullet-list → numbered-clause collision

When a numbered clause contains a **bullet list** and is then followed by numbered
sub-clauses, those sub-clauses must be indented **one tab level deeper than the
bullets**, otherwise the renderer absorbs them into the bullet list (numbering
breaks). This pattern was fixed in ~9 places (e.g. 3.1.2, 3.4.4.4, 5.3.5/5.3.6,
6.1.2.1.1, 6.1.3.1.1.1, 6.4.2.1.1). If you add a clause after an in-clause bullet
list, apply the same +1-tab rule.

## Build & deploy pipeline

- `build_page.py` — the preprocessor (iA Writer Markdown → kramdown-clean).
  Run: `python3 build_page.py <SOURCE>.md > index_body.md`
- `.github/workflows/pages.yml` — GitHub Action. On push to **`master`** it runs
  the preprocessor, prepends Jekyll front matter (`layout: ruleset`), builds with
  Jekyll, and deploys to Pages. (Branch is `master`, not `main`.)
- `_config.yml` — Jekyll config (kramdown, GFM input, `parse_block_html: true`).
- `_layouts/ruleset.html` — the page shell (night-mode toggle, optional PDF link).
- `assets/css/` — the theme CSS (originals, unchanged). `style-ruleset.css` holds
  the counter logic; `style-web-extras.css` holds web-only additions.
- `assets/diagrams/` — the two crease SVGs:
  `diagram-1-crease.svg`, `diagram-2-alternate-crease.svg`
  (all lowercase; there must be exactly ONE `assets/` folder — a capital `Assets/`
  collides on case-insensitive filesystems).

## Local preview

Needs **Ruby 3.x** — macOS system Ruby (2.6) is too old for the `ffi` gem. This
repo pins **3.3.6** via `.ruby-version` (rbenv), matching CI's `ruby-version: 3.3`.

One command — regenerates `index.md` from the source, then serves with live-reload:

```bash
./serve.sh                 # http://localhost:4000
```

`serve.sh` mirrors `.github/workflows/pages.yml` exactly. Equivalent manual steps:

```bash
bundle install
python3 build_page.py EHBA-ruleset-rev230712-IT.md > _body.md
printf '%s\n' '---' 'layout: ruleset' 'title: EHBA — Regolamento Ufficiale (IT)' '---' '' | cat - _body.md > index.md
bundle exec jekyll serve   # http://127.0.0.1:4000
```

`index.md` is a *generated* file (never commit or hand-edit it — see the golden rule).
After editing the source, always preview and check that section numbering is
still correct (spot-check a deeply nested spot like §6.1.3 and a bullet-then-
numbered spot like §3.1.2).

## iA Writer syntax used in the source (do not remove)

- `{{TOC}}` — table of contents (preprocessor expands it for the web).
- `+++` — PDF page breaks (preprocessor strips them for the web).
- `# Title [sectionN]` / `### … [diagramN]` — heading IDs.
- `[Text][sectionN]` — cross-reference links to those IDs.
- `<a name="X.Y.Z"></a>` — inline anchors for clause cross-references
  (e.g. `[`5.4`](#5.4)`). Keep anchors and their `[`n.n`](#n.n)` links in sync.

## Conventions & preferences

- **British English** in prose meta (README etc.); the ruleset body is Italian.
- **README.md is in English** (public entry point); `CHANGELOG_IT.md` is Italian.
- Terminology decisions already made: *Flagranza* (Flagrance), *Incastro*
  (Jamming), *Scooping* kept in English, etc. Stay consistent with existing terms;
  don't silently re-translate established ones.
- Prefer **strict accuracy over speculation**. If a rule reference or a term is
  uncertain, say so and flag it rather than guessing.
- When changing rules to match a newer English revision, keep the change list in
  `CHANGELOG_IT.md` up to date.

## Validation checklist before committing a source edit

1. Numbering still correct after a local `jekyll serve` (CSS counters).
2. No `Assets/` (capital) folder; diagrams under `assets/diagrams/` lowercase.
3. Every `[`n.n`](#n.n)` cross-reference resolves to an existing `<a name>`.
4. Every `[Text][sectionN]`/`[diagramN]` target heading still exists.
5. iA Writer PDF export still renders (the source stays iA-compatible).
